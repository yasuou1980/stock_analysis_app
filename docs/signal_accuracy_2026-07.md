# シグナル精度改善 2026-07: 実測データに基づく診断と対策

## 概要

`results/signals_history.csv` に蓄積された **59 営業日 / 30 銘柄 / 245 件の新規シグナル**を
フォワードリターンで検証し、精度を毀損している構造的パターンを特定した。
遡及シミュレーションでは、採用した 5 つのゲートにより全シグナル区分の勝率が改善する
(トレンドSELL 10日勝率 47.7%→59.3%、逆張りSELL 32.2%→45.2%、逆張りBUYの壊滅トレード 96% 遮断)。

**方法論の要点: 「もっと良い指標を足す」のではなく「負けが実証されている状況で
シグナルを出さない」ことで精度を上げる。** 各ルールは (1) 実測データで抑制群の成績が
明確に悪いこと、(2) 経済合理性が説明できること、の両方を満たす場合のみ採用した。

## 計測方法

- 新規シグナル発生 (onset: 前営業日と異なる BUY/SELL への変化) のみを 1 件と数える
- N 営業日後の終値リターンで判定。SELL は符号反転 (下がれば「勝ち」)
- 期間: 2026-04-09 〜 2026-07-02。v2 ロジック (2026-06-09〜) の分は版別に確認
- onset 時の RSI / 乖離率 / 直近5日リターン / 20日高値からの下落率 / 商品クラスで
  バケット分析し、勝率が構造的に低いバケットを特定

## 診断: 精度を失っていた 4 つの構造的パターン

### 1. インバース型レバレッジETF (SOXS/SQQQ/SPXS) の BUY — 最大の毀損源

逆張り BUY 33 件中 **31 件がインバース型レバETF** で、10日後勝率 19.4%・平均 -13.9%。
トレンドフォロー BUY でも 2 件発生し全敗 (平均 -8.2%/5日)。

3倍インバース商品は日次リバランスによる構造的減価があり、上昇相場では
「常に売られすぎ」に見えるため逆張りロジックを恒常的に誤作動させる。
また急騰後の順張り BUY は「暴落がさらに続くこと」に賭ける行為で、期待値が負。

### 2. ロング型レバレッジETF (TQQQ/SOXL/UPRO等) の トレンドSELL

5日後勝率 **7.1%** (平均 -12.2%)、10日後勝率 18.2% (平均 -18.0%)。

SELL には EMA50 割れ確認が必須のため、3倍ETFでは確認が取れた時点で下げが出切っており、
指数が 2〜3% 戻るだけで 6〜9% の反発に轢かれる。2026-06-05 の SOXL SELL は
10日で **-64.8%** (SOXL が +65% 反発) の最悪例。

### 3. 急落直後の SELL (投げ売りの底で売る)

直近5日で -12% 超の急落後に出た SELL: トレンド系 勝率 21.4% (5日)、
逆張り系は -10% 超の急落後で勝率 **12.5%** (5日) / 16.7% (10日)・平均 -17.3%。
v2 ロジック期間 (6/9以降) でもトレンドSELL の 5日勝率は 15.8% で、
「構造が壊れてから売る」設計が短期の反発に食われ続けていた。

### 4. 逆張り SELL の概念矛盾 (「売られた後に売る」)

逆張り SELL は本来「過熱からの反落」を売るはずが、実際の発火時状態は
RSI 28〜51・乖離率マイナスが多数 (トリガーが「過熱圏からの脱出後」+「直近安値割れ」
のため、発火時にはすでに下げた後)。onset 時状態別の 10日後勝率:

| onset 時の状態 | 件数 | 勝率 | 平均 |
|---|---|---|---|
| RSI > 70 (過熱のまま売り) | 15 | 13.3% | -16.1% |
| 乖離率 > +15% (踏み上げ帯) | 16 | 12.5% | -24.4% |
| 急落直後 (5日 -10% 超) | 12 | 16.7% | -17.3% |
| RSI≥50 かつ 乖離率 -3〜+15% かつ 急落直後でない | 31 | **45.2%** | -2.3% |

勝てる状態は「まだ強さが残っている (RSI≥50・乖離が平均近辺〜やや上) が反落し始めた」
ケースに集中している。

### 対照: 機能しているシグナル (触らない)

- **トレンドフォロー BUY (現物・非レバ銘柄)**: 10日後勝率 **78.6%**・平均 +8.6% (n=28)。本システムの主力
- **トレンドフォロー SELL (インバースETF)**: 10日後勝率 75%・平均 +14.3% (減価継続の売り)

## 採用した改善 (5 ルール)

`backtester.compute_signal_gates()` に実装。商品クラスは `config.toml` の
`[ticker_classes]` で宣言し、未宣言の銘柄は plain (現物扱い)。

| # | ルール | 根拠 (抑制される群の実測) |
|---|---|---|
| G1 | インバース型レバETFは **BUY 禁止** (全戦略) | 33件 勝率 0〜19%・平均 -13.9% (10日) |
| G2 | ロング型レバETFは **トレンドSELL 禁止** (手仕舞いは ATR トレイリングストップが担当) | 14件 勝率 7〜18%・平均 -12〜-18% |
| G3 | 非インバース銘柄は 5日 -12% 超の**急落直後の トレンドSELL 禁止** | 13-14件 勝率 21〜38%・平均 -6〜-7.6% |
| G4 | ロング型レバETFは **逆張りSELL 禁止** | 23-25件 勝率 20〜26%・平均 -16% (10日) |
| G5 | 逆張りSELL は **RSI≥50 かつ 乖離率 -3〜+15% かつ 5日 -10% 以内**のみ許可 | 抑制群 59件 勝率 25.4%・平均 -12.1% (10日) |

### 遡及シミュレーション (過去 245 onset に適用した場合)

| シグナル | 指標 | 現状 | 適用後 |
|---|---|---|---|
| トレンドフォロー BUY | 10日勝率 / 平均 | 67.4% / +5.4% | **70.5% / +5.9%** |
| トレンドフォロー SELL | 5日勝率 / 平均 | 32.1% / -1.7% | **42.4% / +3.0%** |
| トレンドフォロー SELL | 10日勝率 / 平均 | 47.7% / -0.2% | **59.3% / +6.5%** |
| トレンドフォロー SELL | 20日勝率 / 平均 | 45.0% / +7.7% | **52.9% / +15.1%** |
| 逆張り BUY | 件数 | 33 件 (勝率 21%) | 2 件 (毒性トレード 96% 遮断) |
| 逆張り SELL | 10日勝率 / 平均 | 32.2% / -8.7% | **45.2% / -2.2%** |
| 逆張り SELL | 20日勝率 / 平均 | 35.7% / -11.8% | **53.6% / +0.3%** |

シグナル件数は減る (SELL は約 4 割に絞られる) が、「出したシグナルが当たる確率」を
優先する設計判断。抑制された状況の手仕舞いはバックテスター側の
ATR ストップ・トレイリングストップ・タイムストップが引き続き担当する。

## 検証したが不採用にしたもの (重要)

- **乖離率 +25〜30% 超のトレンドBUY 抑制 (過熱飛びつき防止)**: 直感に反し、
  抑制されるはずの群の方が高成績だった (10日勝率 77.3%・平均 +10.2%)。
  強い過伸長はこの銘柄群ではモメンタム継続シグナルであり、抑制すると精度が**下がる**。
  6/25 の MU BUY (-19.6%) のような事例は目立つが例外側だった
- **RSI<33 での トレンドSELL 全面禁止**: 抑制群の 10日勝率 66.7% と勝ち筋
  (暴落継続・インバース減価) も殺してしまう。急落クールダウン (G3) と
  商品クラス (G2) の組み合わせの方が判別精度が高い

「もっともらしい改善案」の半分は実測で逆効果と判明した。**今後の変更も必ず
履歴データで抑制群/残存群を分けて検証してから入れるべき**で、そのための
計測基盤を今回強化した (下記)。

## 計測基盤の強化 (今後の改善ループのため)

1. **特徴量の蓄積**: 日次バッチが `score` (シグナルスコア)・`adx`・`ret_5d`・
   `ticker_class` を `signals_history.csv` と SQLite に保存するようにした。
   これまで RSI と乖離率しか記録がなく検証できる仮説が限られていた
2. **ロジック版別レポート**: `performance_report.txt` に版別 (v1/v2/v3) の
   勝率集計を追加。改善が本当に効いたかを変更後のシグナルだけで確認できる
   (`signal_tracker.LOGIC_VERSIONS` にロジック変更日を追記して運用)
3. **回帰テスト**: `tests/` にゲートの単体テストと 3 戦略の統合テストを追加
   (`python -m pytest tests/`)

## 制約と注意

- サンプルは 59 営業日・245 onset と小さく、期間は強い上昇相場 + 急落 2 回に偏る。
  各ルールは効果量が大きく (勝率差 20pt 超) 経済合理性があるものに限定したが、
  **地合いが変われば再検証が必要** (特に本格的な下落相場では G2/G5 の見直し余地)
- 遡及シミュレーションは「同じ相場で同じ onset が出た」前提の近似。
  厳密な評価は v3 稼働後の版別レポートで行う
- `scriptable/StockSignal.js` にも同一ゲートを移植済み。ロジック変更時は
  Python / JS / `config.toml` の 3 点を同期すること

## 今後の改善候補 (優先順)

1. **v3 の実測検証** (2〜4 週間後): 版別レポートで v2→v3 の勝率変化を確認し、
   効かないルールは撤回する
2. **スコア閾値の実測較正**: `score` の蓄積が進んだら「スコアマージン vs 勝率」を
   分析し、確信度の低い閾値ぎりぎりのシグナルを HOLD に落とす
   (現在は閾値超えなら一律 BUY/SELL)
3. **地合いフィルタ**: SPY/QQQ のトレンド状態で個別シグナルをゲートする
   (6/5 のような暴落日に 20 件超の SELL が一斉発火して全滅する問題への対処)
4. **決算イベント回避**: 決算発表直前の新規シグナルを保留する
   (6/25 前後の MU/AMAT のような決算ギャップによる即死の回避)
5. **ウォークフォワード検証基盤**: 数年分の OHLCV をローカル取得して
   「過去 N 日で較正 → 次の M 日で検証」を回し、期間バイアスを除去する
6. **逆張り BUY の再設計**: 現在は事実上停止中 (G1 + v2 フィルタで onset 0 件)。
   再開するなら「非レバ大型株の上昇トレンド中の浅い押し目」に限定した
   別トリガーとして設計し直す
